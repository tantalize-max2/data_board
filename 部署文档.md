# 百日攻坚"夏收"行动标准化穿透部署 — 部署文档

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | 夏收行动部署看板（data_board） |
| 技术栈 | Python 3.11 + FastAPI + MySQL 8.0 + Nginx |
| 部署方式 | Docker Compose（三容器编排） |
| 适用场景 | 150 并发用户的内部看板系统 |
| 默认端口 | Nginx 80（对外）、App 8000（内部）、MySQL 3306（内部） |

### 架构图

```
用户浏览器
    │
    ▼
┌──────────┐     ┌──────────────────┐     ┌──────────┐
│  Nginx   │────▶│  FastAPI App     │────▶│  MySQL   │
│  :80     │     │  (uvicorn :8000) │     │  :3306   │
│  反向代理 │     │  4 workers       │     │  8.0     │
└──────────┘     └──────────────────┘     └──────────┘
                  静态文件直出              数据持久化卷
```

### 目录结构

```
data_board/
├── server.py              # FastAPI 后端（路由、业务逻辑）
├── db.py                  # 数据库连接池（环境变量配置）
├── static/                # 前端静态文件
│   ├── index.html         # 主看板页面
│   ├── admin.html         # 管理后台页面
│   ├── css/               # 样式文件
│   ├── js/                # 脚本文件
│   └── background.png     # 背景图
├── Dockerfile             # 应用镜像构建
├── docker-compose.yml     # 三容器编排（MySQL + App + Nginx）
├── nginx.conf             # Nginx 反向代理配置
├── init-db.sql            # 数据库自动初始化脚本
├── requirements.txt       # Python 依赖
├── .env.example           # 环境变量模板
├── .dockerignore          # Docker 构建排除
└── 部署文档.md            # 本文档
```

## 三、快速部署（Docker Compose）

### 第一步：拉取代码

```bash
git clone https://github.com/tantalize-max2/data_board.git
cd data_board
```

### 第二步：配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑配置（务必修改密码）
vi .env
```

`.env` 文件内容：

```ini
# 数据库配置
DB_HOST=mysql
DB_PORT=3306
DB_USER=root
DB_PASSWORD=YourStrongPassword123!    # ← 必须修改
DB_DATABASE=xiashou2

# 连接池（150并发建议）
DB_POOL_MIN=4
DB_POOL_MAX=20
DB_POOL_MAX_CONN=50

# Uvicorn worker 数（等于 CPU 核心数）
UVICORN_WORKERS=4
```

### 第三步：拉取 MySQL 镜像

```bash
docker pull mysql:8.0
docker pull nginx:1.27-alpine
```

### 第四步：启动服务

```bash
# 构建并启动全部容器（首次构建镜像）
docker compose up -d --build

# 查看启动状态
docker compose ps
```

正常输出：

```
NAME              STATUS                   PORTS
xiashou-mysql     Up (healthy)             127.0.0.1:3306->3306/tcp
xiashou-app       Up (healthy)             127.0.0.1:8000->8000/tcp
xiashou-nginx     Up                       0.0.0.0:80->80/tcp
```

### 第五步：验证

```bash
# 测试 API
curl http://localhost/api/roles

# 浏览器访问
http://你的服务器IP/
```

默认管理员账号：`admin` / 密码：`123456`（首次登录后请立即修改）

## 四、数据库初始化说明

`init-db.sql` 会在 MySQL 容器**首次启动**时自动执行（挂载到 `/docker-entrypoint-initdb.d/`）。

自动创建的内容：

| 表名 | 说明 |
|------|------|
| `battles` | 6 大战役（一号工程战、百万加装战等） |
| `warzones` | 4 大战区（公众、商业、校园、行业） |
| `users` | 用户表（含默认管理员） |
| `role_access` | 角色权限矩阵 |
| `deployment_records` | 部署记录表（144 条业务数据需手动导入） |

> **注意**：`init-db.sql` 只在 MySQL 数据卷为空（首次创建）时执行。如果数据卷已存在，修改 `init-db.sql` 后需删除数据卷重建：
> ```bash
> docker compose down -v   # -v 删除数据卷
> docker compose up -d --build
> ```

## 五、导入业务数据

`init-db.sql` 仅包含表结构和种子数据（战役/战区/管理员）。完整的 144 条部署记录需要通过 Excel 导入。

### 方式一：命令行导入（推荐）

```bash
# 1. 将 Excel 数据导出为 CSV 或直接用 Python 脚本导入
# 2. 进入 MySQL 容器
docker exec -it xiashou-mysql mysql -u root -p xiashou2

# 3. 检查现有数据
SELECT COUNT(*) FROM deployment_records;
```

### 方式二：Python 脚本导入

```bash
# 在项目根目录执行（需本地 Python 环境）
pip install pandas pymysql openpyxl

# 运行导入脚本（参考 .temp/import_excel.py）
python import_excel.py
```

### 验证导入结果

```sql
-- 总记录数应为 144
SELECT COUNT(*) FROM deployment_records;

-- 各战区分布
SELECT warzone_name, COUNT(*) FROM deployment_records GROUP BY warzone_name;
-- 预期：公众86 商客20 校园6 行业32

-- 各战役分布
SELECT battle_name, COUNT(*) FROM deployment_records GROUP BY battle_name;
-- 预期：一号工程32 双线29 百万32 细分27 掘金14 高价值10
```

## 六、端口与网络说明

| 端口 | 服务 | 对外暴露 | 说明 |
|------|------|---------|------|
| 80 | Nginx | ✅ 是 | 唯一对外端口，所有请求入口 |
| 8000 | FastAPI (uvicorn) | ❌ 仅 127.0.0.1 | Nginx 反向代理到此处 |
| 3306 | MySQL | ❌ 仅 127.0.0.1 | 仅本机容器间通信 |

> 安全设计：App 和 MySQL 只绑定 `127.0.0.1`，外部无法直连，所有流量经过 Nginx。

## 七、常用运维命令

### 启停管理

```bash
# 启动全部服务
docker compose up -d

# 停止全部服务（保留数据）
docker compose down

# 停止并删除数据卷（⚠️ 清空数据库）
docker compose down -v

# 重启单个服务
docker compose restart app
docker compose restart nginx

# 重新构建镜像（代码更新后）
docker compose up -d --build app
```

### 日志查看

```bash
# 全部日志
docker compose logs -f

# 单个服务日志
docker compose logs -f app
docker compose logs -f mysql
docker compose logs -f nginx

# 最近 100 行
docker compose logs --tail 100 app
```

### 进入容器

```bash
# 进入应用容器
docker exec -it xiashou-app bash

# 进入 MySQL
docker exec -it xiashou-mysql mysql -u root -p xiashou2

# 进入 Nginx
docker exec -it xiashou-nginx sh
```

### 健康检查

```bash
# 检查容器健康状态
docker compose ps

# 手动测试健康检查
docker inspect --format='{{.State.Health.Status}}' xiashou-app

# 测试 API 响应
curl -s http://localhost/api/roles | head -c 200
```

## 八、本地开发部署（非 Docker）

适用于开发调试场景。

### 前置条件

- Python 3.11+
- MySQL 8.0（本地或远程）

### 步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动 MySQL（本地已安装可跳过）
# 确保本地 MySQL 运行在 127.0.0.1:3306，root/root，数据库 xiashou2

# 3. 初始化数据库（如未初始化）
mysql -u root -p < init-db.sql

# 4. 启动服务（单进程，调试模式）
uvicorn server:app --host 127.0.0.1 --port 8001 --reload

# 5. 浏览器访问
http://127.0.0.1:8001/
```

### 本地开发默认配置（db.py）

```python
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=root
DB_DATABASE=xiashou2
```

> 本地开发无需 `.env` 文件，`db.py` 有默认值。生产环境通过环境变量覆盖。

## 九、性能调优

### 连接池参数（`.env`）

| 参数 | 默认值 | 150并发建议 | 说明 |
|------|--------|------------|------|
| DB_POOL_MIN | 4 | 4 | 最小空闲连接 |
| DB_POOL_MAX | 20 | 20 | 最大缓存连接 |
| DB_POOL_MAX_CONN | 50 | 50 | 最大物理连接数 |

### uvicorn workers

```bash
# .env 中设置
UVICORN_WORKERS=4    # 一般等于 CPU 核心数

# 或修改 docker-compose.yml 中 CMD 的 --workers 参数
```

### MySQL 调优（docker-compose.yml）

```yaml
command: >
  --character-set-server=utf8mb4
  --collation-server=utf8mb4_unicode_ci
  --max-connections=200          # 可根据并发调大
  --innodb-buffer-pool-size=256M # 可根据内存调大
```

# 恢复全库

## 十一、安全注意事项

1. **修改默认密码**：部署后立即在管理后台修改管理员密码
2. **`.env` 不入库**：`.env` 文件含密码，已在 `.dockerignore` 中排除
3. **MySQL 不对外**：仅绑定 `127.0.0.1:3306`，外部无法直连
4. **Nginx 安全头**：已配置 `X-Frame-Options` 和 `X-Content-Type-Options`
5. **HTTPS**：生产环境建议在 Nginx 配置 SSL 证书

### 配置 HTTPS（可选）

```nginx
# nginx.conf 增加 443 端口
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # ... 其余配置同 80 端口
}

# 80 端口重定向到 443
server {
    listen 80;
    return 301 https://$host$request_uri;
}
```

## 十二、故障排查

### 问题：容器启动后无法访问

```bash
# 1. 检查容器状态
docker compose ps

# 2. 检查是否所有容器都 Up (healthy)
# 如果 app 或 mysql 不是 healthy，查看日志
docker compose logs app
docker compose logs mysql

# 3. 检查端口占用
sudo netstat -tlnp | grep :80
sudo netstat -tlnp | grep :8000
```

### 问题：MySQL 连接失败

```bash
# 1. 检查 MySQL 健康状态
docker inspect --format='{{.State.Health.Status}}' xiashou-mysql

# 2. 检查密码是否正确
docker exec -it xiashou-mysql mysql -u root -p

# 3. 检查 .env 中 DB_PASSWORD 与 docker-compose.yml 是否一致
```

### 问题：pip 安装失败（零宽字符）

```bash
# Dockerfile 中已内置清理步骤，如仍失败可手动清理
docker exec -it xiashou-app bash
python -c "import re;c=open('requirements.txt',encoding='utf-8').read();open('requirements.txt','w').write(re.sub(r'[\u200b-\u200f\ufeff]','',c))"
pip install -r requirements.txt
```

### 问题：数据卷冲突（修改 init-db.sql 不生效）

```bash
# MySQL 只在首次启动时执行 init-db.sql
# 修改后需要删除数据卷重建
docker compose down -v        # ⚠️ 会删除所有数据
docker compose up -d --build
```

### 问题：查看实时日志定位错误

```bash
# 实时跟踪应用日志
docker compose logs -f app

# 过滤错误信息
docker compose logs app 2>&1 | grep -i error
```

## 十三、更新部署

代码更新后重新部署：

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 重新构建并启动（数据库数据不丢失）
docker compose up -d --build app

# 3. 验证
docker compose ps

```

> 只有 `app` 容器需要重建，MySQL 和 Nginx 不受影响。
